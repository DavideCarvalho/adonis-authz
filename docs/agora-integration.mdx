All four features below are **opt-in** and **default-OFF**. They read well-known
`globalThis` slots structurally and never add a hard dependency, so authz keeps
working standalone with unchanged default behavior.

## Tenant auto-scope

When a permission check gets no explicit tenant and your `tenant` resolver
yields none, default the tenant to the active Agora context's `tenantId`.

```ts title="config/authz.ts"
import { defineConfig, stores } from '@adonis-agora/authz'

export default defineConfig({
  default: 'lucid',
  stores: { lucid: stores.lucid() },
  // 'context' uses the built-in tenantFromContext() reader.
  resolveTenant: 'context',
  // …or supply your own:
  // resolveTenant: () => myTenant(),
})
```

Precedence: explicit `scope` arg → `tenant` resolver → `resolveTenant`. With
`resolveTenant` unset (the default), no context means the global `''` scope, as
before.

## Global-role bridge

Bridge an auth provider's **global roles** (read structurally from the Agora
context store under `globalRoles`, written by authkit) into authz checks at
check time — no DB seeding.

```ts
defineConfig({
  // A user holding one of these global roles is allowed everything.
  superAdminRoles: ['platform:super'],
  // Global role → permissions/wildcards, unioned into permission checks.
  globalRoleGrants: {
    auditor: ['audit.*'],
  },
})
```

A super-admin global role short-circuits `can` and `hasRole` to allow.
Otherwise, `globalRoleGrants` for the user's active global roles are unioned
into the permission match before consulting the store.

## Event-driven provisioning

`defineAuthzProvisioning` subscribes to authkit's diagnostics events (via the
**optional** `@adonis-agora/diagnostics` peer) and runs config-mapped actions
against the store. It is best-effort — actions never throw into the host, and a
missing diagnostics package is a no-op.

```ts title="start/authz_provisioning.ts"
import { defineAuthzProvisioning } from '@adonis-agora/authz/provisioning'

const authz = await app.container.make(AuthzService)

await defineAuthzProvisioning({
  store: authz.store,
  on: {
    'organization.created': (ev, store) =>
      store.assignRole(ev.metadata.userRef, 'org:owner', { tenantId: ev.metadata.orgId }),
    'member.added': (ev, store) =>
      store.assignRole(ev.metadata.userRef, 'org:member', { tenantId: ev.metadata.orgId }),
  },
})
```

Keys are the bare authkit event type (subscribed on channels
`agora:authkit:<type>`). The call returns a handle with `.stop()` to
unsubscribe.

## The `/authz/can` endpoint

`registerCanEndpoint` registers a route that answers a permission check for the
active user. Optional peer: `@adonisjs/core` (router types — already a peer).

```ts title="start/routes.ts"
import router from '@adonisjs/core/services/router'
import { registerCanEndpoint } from '@adonis-agora/authz/http'

const authz = await app.container.make(AuthzService)
registerCanEndpoint(router, { service: authz }) // default path: /authz/can
```

<Callout type="info">
  **Contract (the frontend depends on this exactly):**

  - Request JSON: `{ "permission": string, "resource"?: string }`
  - Response JSON: `{ "allowed": boolean }`

  An anonymous/unmappable user or a missing `permission` yields
  `{ "allowed": false }`.
</Callout>

Override the path and user resolution as needed:

```ts
registerCanEndpoint(router, {
  service: authz,
  path: '/api/authz/can',
  resolveUser: (ctx) => ctx.auth?.user,
})
```
