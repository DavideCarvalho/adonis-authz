---
title: Ace commands
description: Manage roles and permissions from the command line.
icon: Terminal
---

`configure` registers the commands barrel. All commands boot the app and use the
configured store.

## `authz:make-role`

```sh
node ace authz:make-role editor
```

Create a role (idempotent).

## `authz:make-permission`

```sh
node ace authz:make-permission posts.edit
```

Create a permission. Names may be dotted/wildcard (e.g. `posts.*`).

## `authz:grant`

```sh
node ace authz:grant editor posts.edit
```

Grant a permission to a role (creating either by name as needed).

## `authz:assign`

```sh
node ace authz:assign editor 42
node ace authz:assign editor 42 --type=admin --tenant=acme
```

Assign a role to a user. `--type` sets the polymorphic user type (default
`user`); `--tenant` scopes the assignment (omit for the global scope).

## `authz:list`

```sh
node ace authz:list
```

List every role (with its permissions) and every permission.

## `authz:sync`

```sh
node ace authz:sync
```

Seed roles and permissions from the `catalog` declared in `config/authz.ts`.
Idempotent — only missing rows are created/attached.

```ts title="config/authz.ts"
defineConfig({
  catalog: {
    permissions: ['system.manage'],
    roles: {
      editor: ['posts.*'],
      viewer: ['posts.view'],
    },
  },
})
```
